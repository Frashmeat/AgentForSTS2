# 2026-04-09 设置页独立路由改造记录

## 1. 目标

把工作区内原先通过右侧抽屉打开的设置界面改成独立页面，同时保持现有设置读取、路径检测和保存逻辑不变。

## 2. 本次改动范围

- `frontend/src/App.tsx`
- `frontend/src/components/SettingsPanel.tsx`
- `frontend/src/pages/SettingsPage.tsx`
- `frontend/tests/settingsPageRoute.test.ts`

## 3. 实现结果

- 工作区设置入口改为跳转 `/settings`
- `/settings` 页面支持携带 `returnTo` 参数返回原工作区 tab
- 原 `SettingsPanel` 设置表单逻辑保留，仅新增页面化承载模式
- `Claude API` 模式下新增 Claude 模型输入框，避免文本能力检测因 `llm.model` 为空而误判失败
- `Claude API` 模式下，文本任务和代码代理都会跟随同一套 `model/api_key/base_url`
- 本机执行不可用时，执行方式弹窗会显示缺失配置原因，并区分文本 / 代码代理 / 图像能力
- 不影响 `/auth/*`、`/me/*` 路由

## 4. 定向验证建议

建议执行：

```powershell
node --test .\frontend\tests\settingsDetectProgress.test.js
node --test .\frontend\tests\settingsSaveFailure.test.js
node --test .\frontend\tests\settingsPageRoute.test.ts
node .\frontend\tests\executionModeDialog.test.ts
node .\frontend\tests\configApi.test.ts
python -m pytest backend/tests/test_local_ai_capability_status.py -q
```

验证目的：

- 确认设置页仍保留路径检测轮询与中断逻辑
- 确认设置保存失败时仍会复位按钮状态并显示错误
- 确认前端已接入独立设置路由，且不再依赖 `settingsOpen` 抽屉状态

## 5. 风险提醒

- 当前设置表单仍位于 `SettingsPanel.tsx`，后续若继续语义收口，可再拆成更明确的 `SettingsForm` 组件。
- 若未来需要从非工作区页面进入设置，`returnTo` 规则需要补充更严格的来源策略。
