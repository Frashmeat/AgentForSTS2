# 2026-04-07 平台页面视觉统一验收记录

## 1. 目标

把认证页与用户中心页的视觉语言收口到工作站控制台体系，避免 `/auth/*`、`/me/*` 继续维持旧的白卡片工具页风格。

## 2. 本次实现范围

- 新增 `frontend/src/components/platform/PlatformPageShell.tsx`
- 认证页：
  - `frontend/src/features/auth/LoginPage.tsx`
  - `frontend/src/features/auth/RegisterPage.tsx`
  - `frontend/src/features/auth/ForgotPasswordPage.tsx`
  - `frontend/src/features/auth/ResetPasswordPage.tsx`
  - `frontend/src/features/auth/VerifyEmailPage.tsx`
- 用户中心页：
  - `frontend/src/features/user-center/page.tsx`
  - `frontend/src/features/user-center/job-detail-page.tsx`
  - `frontend/src/features/user-center/historyList.tsx`
  - `frontend/src/features/user-center/quotaCard.tsx`
  - `frontend/src/features/user-center/refundSummary.tsx`
- 共享样式：
  - `frontend/src/index.css`

## 3. 改造原则

- 复用工作站现有的紫蓝主题 token、玻璃面板和高亮色，不另起一套主题
- 不把工作站左侧导航直接搬到认证页与用户中心
- 不改登录、注册、邮箱验证、任务读取、返还摘要等业务逻辑
- 保留返回首页与用户中心导航链路

## 4. 最小验证

### 4.1 平台页统一壳层回归

执行：

```powershell
node --test .\frontend\tests\platformUnifiedShell.test.ts
```

覆盖点：

- 认证页与用户中心页复用 `PlatformPageShell`
- 平台页面样式复用工作站主题变量

### 4.2 认证页定向 TypeScript 编译

执行：

```powershell
node .\frontend\node_modules\typescript\bin\tsc --noEmit --target ES2020 --module ESNext --moduleResolution bundler --jsx react-jsx --allowImportingTsExtensions --skipLibCheck src/features/auth/AuthHomeLink.tsx src/features/auth/LoginPage.tsx src/features/auth/RegisterPage.tsx src/features/auth/ForgotPasswordPage.tsx src/features/auth/ResetPasswordPage.tsx src/features/auth/VerifyEmailPage.tsx
```

覆盖点：

- 新增共享壳层与认证页引用关系正确
- 认证页局部改造未引入 TypeScript 编译错误

## 5. 结果结论

- 认证页与用户中心页已经切换到和工作站一致的主题语言
- 登录页、注册页、找回密码页、重置密码页、验证邮箱页都不再是孤立白卡片样式
- 用户中心首页与任务详情页的标题区、信息卡片、统计卡片与列表交互已和工作站主题收口
- 本次仍未引入新的 UI 框架，也未修改业务接口和数据流
