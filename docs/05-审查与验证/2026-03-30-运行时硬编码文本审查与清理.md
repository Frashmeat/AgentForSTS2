# 运行时硬编码文本审查与清理

日期：2026-03-30

## 审查结论

本轮审查聚焦“生产代码中作为运行时真源的硬编码文本”，不包含：

- 注释
- docstring
- 测试断言文本
- 协议字段名、路由路径、枚举值、环境变量名

审查后确认仍有三类运行时硬编码文本需要迁移：

1. 已有 bundle 对应的 LLM prompt 仍保留 Python 内联 fallback
2. `workflow` / `batch_workflow` / `build_deploy` / `config_router` / `project_utils` 中散落的用户可见消息
3. `image.prompt_adapter` 中作为 guide / fallback 真源的默认文本

### 3. 迁移用户可见消息与模板

以下模块已把主要运行时消息 / 模板迁移到 `.md`：

- `backend/routers/workflow.py`
- `backend/routers/batch_workflow.py`
- `backend/routers/build_deploy.py`
- `backend/routers/config_router.py`
- `backend/project_utils.py`

迁移内容包括：

- 审批流程提示
- 项目初始化提示

- 构建 / 部署提示

- STS2 / Godot 路径探测说明
- 模板缺失错误说明

### 4. 收口 image prompt 默认文案

`backend/image/prompt_adapter.py` 已不再以内联默认 guide 文案作为真源，改为从 `image.md` 读取：

- provider lang
- formula
- rules
- example
- negative example
- transparent rule
- fallback warning
- fallback suffix / negative prompt
