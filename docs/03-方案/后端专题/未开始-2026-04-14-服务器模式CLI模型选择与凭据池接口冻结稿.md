## 1. 冻结目标

本稿只解决 4 件事：

1. 冻结普通用户侧服务器配置接口
2. 冻结管理员第一批凭据管理接口
3. 冻结接口字段命名与最小响应结构
4. 明确哪些信息允许暴露给普通用户，哪些只允许管理员侧使用

本稿不解决：

- 数据库 migration 细节
- ORM 模型实现
- 管理员鉴权代码
- 前端弹窗和设置页 UI 结构
- 自动切换执行逻辑实现

## 2. 全局约束

### 2.1 角色边界

- 普通用户接口只暴露 `CLI + 模型` 级别的能力，不暴露 provider、key、base URL 或单把凭据健康细节。
- 管理员接口负责维护真实服务器凭据，但必须建立在后续管理员权限边界补齐之后。
- `web` 仍是唯一对外平台入口，服务器侧 `workstation` 只接收内部执行请求。

### 2.2 命名口径

当前冻结以下命名：

- 用户可选组合：`execution_profile`
- 后台真实凭据：`server_credential`
- 用户默认服务器配置：`server_preferences`
- 执行命中快照：仍通过 `job` 与 `ai_execution` 表达，不额外在接口层发明新概念

### 2.3 HTTP 约束

- 用户侧接口统一挂在 `/api/platform/*` 或 `/api/me/*`
- 管理员侧接口统一挂在 `/api/admin/platform/*`
- 第一版统一使用 JSON
- 错误响应延续现有平台错误风格，不在本稿单独发明第二套 envelope

## 3. 普通用户侧接口冻结

## 3.1 `GET /api/platform/execution-profiles`

### 用途

- 运行时服务器配置选择弹窗
- 设置页 `服务器模式` Tab 中的“更换配置”

### 请求

- 无请求体
- 暂不支持筛选参数

### 响应

```json
{
  "items": [
    {
      "id": 1,
      "display_name": "Codex CLI / gpt-5.4",
      "agent_backend": "codex",
      "model": "gpt-5.4",
      "description": "适合复杂代码修改，默认推荐。",
      "recommended": true,
      "available": true
    }
  ]
}
```

### 字段冻结

- `id`
- `display_name`
- `agent_backend`
- `model`
- `description`
- `recommended`
- `available`

### 返回规则冻结

- 只返回 `enabled = true` 的 `execution_profile`
- `available` 由后端根据同组合下是否存在健康可用凭据综合判定
- 默认排序规则冻结为：
  1. `recommended desc`
  2. `sort_order asc`
  3. `id asc`

### 明确不返回

- `provider`
- `credential_count`
- `base_url`
- `health_status`
- 任何 key 相关字段

## 3.2 `GET /api/me/server-preferences`

### 用途

- 设置页读取当前默认服务器配置
- 运行时判断是否可直接复用默认服务器配置

### 请求

- 无请求体
- 要求当前用户已登录

### 响应

已设置默认值时：

```json
{
  "default_execution_profile_id": 1,
  "display_name": "Codex CLI / gpt-5.4",
  "agent_backend": "codex",
  "model": "gpt-5.4",
  "available": true,
  "updated_at": "2026-04-13T16:20:00Z"
}
```

未设置默认值时：

```json
{
  "default_execution_profile_id": null,
  "display_name": "",
  "agent_backend": "",
  "model": "",
  "available": false,
  "updated_at": null
}
```

### 字段冻结

- `default_execution_profile_id`
- `display_name`
- `agent_backend`
- `model`
- `available`
- `updated_at`

### 返回规则冻结

- 若用户未登录，返回认证失败
- 若用户已登录但未设置默认值，返回空态对象，不返回 404
- 若默认配置已下线或暂不可用：
  - 保留 `default_execution_profile_id`
  - `available = false`
  - 仍返回最后一次保存的展示信息

## 3.3 `PUT /api/me/server-preferences`

### 用途

- 设置用户默认服务器配置
- 清除用户默认服务器配置

### 请求

设置默认值：

```json
{
  "default_execution_profile_id": 1
}
```

清除默认值：

```json
{
  "default_execution_profile_id": null
}
```

### 响应

成功后统一返回与 `GET /api/me/server-preferences` 一致的结构。

### 校验规则冻结

- `default_execution_profile_id` 若不为空，必须指向存在的 `execution_profile`
- 指向的 `execution_profile` 必须 `enabled = true`
- 即使该组合当前 `available = false`，第一版也不建议允许用户把它保存成默认值

## 4. 管理员侧接口冻结

## 4.1 `GET /api/admin/platform/server-credentials`

### 用途

- 查看服务器凭据列表
- 允许按组合过滤

### 请求参数

- `execution_profile_id`：可选

### 响应

```json
{
  "items": [
    {
      "id": 101,
      "execution_profile_id": 1,
      "provider": "openai",
      "auth_type": "api_key",
      "label": "openai-main-a",
      "base_url": "https://api.openai.com/v1",
      "priority": 10,
      "enabled": true,
      "health_status": "healthy",
      "last_checked_at": "2026-04-14T09:00:00Z",
      "last_error_code": "",
      "last_error_message": ""
    }
  ]
}
```

### 字段冻结

- `id`
- `execution_profile_id`
- `provider`
- `auth_type`
- `label`
- `base_url`
- `priority`
- `enabled`
- `health_status`
- `last_checked_at`
- `last_error_code`
- `last_error_message`

### 明确不返回

- `credential_ciphertext`
- `secret_ciphertext`
- 原始 key

## 4.2 `POST /api/admin/platform/server-credentials`

### 用途

- 新增服务器凭据

### 请求

```json
{
  "execution_profile_id": 1,
  "provider": "openai",
  "auth_type": "api_key",
  "credential": "sk-***",
  "secret": "",
  "base_url": "https://api.openai.com/v1",
  "label": "openai-main-a",
  "priority": 10,
  "enabled": true
}
```

### 响应

- 返回脱敏后的凭据元数据
- 不回显原始 `credential` / `secret`

### 校验规则冻结

- `execution_profile_id` 必须存在
- `provider`、`auth_type` 取值必须落在后续冻结枚举内
- `credential` 必填
- `priority` 为整数

## 4.3 `POST /api/admin/platform/server-credentials/{id}/health-check`

### 用途

- 手动检测单个服务器凭据

### 响应

```json
{
  "credential_id": 101,
  "health_status": "healthy",
  "error_code": "",
  "error_message": "",
  "checked_at": "2026-04-14T09:05:00Z"
}
```

### 字段冻结

- `credential_id`
- `health_status`
- `error_code`
- `error_message`
- `checked_at`

### 第一版健康状态枚举冻结

- `healthy`
- `degraded`
- `rate_limited`
- `auth_failed`
- `quota_exhausted`
- `disabled`

## 5. 接口与数据模型的直接映射

本稿先冻结接口层与模型层的最小映射关系：

- `GET /api/platform/execution-profiles`
  - 读取 `execution_profiles`

- `GET /api/me/server-preferences`
- `PUT /api/me/server-preferences`
  - 读取 / 写入 `user_platform_preferences`
  - 联查 `execution_profiles`

- `GET /api/admin/platform/server-credentials`
- `POST /api/admin/platform/server-credentials`
- `POST /api/admin/platform/server-credentials/{id}/health-check`
  - 读取 / 写入 `server_credentials`
  - 检测结果落 `credential_health_checks`

## 6. 与任务主链的字段落点冻结

当前冻结如下：

- `job`
  - `selected_execution_profile_id`
  - `selected_agent_backend`
  - `selected_model`

- `ai_execution`
  - `provider`
  - `model`
  - `credential_ref`
  - `retry_attempt`
  - `switched_credential`

当前不冻结到 `job_item`：

- `execution_profile_id`

原因：第一版用户是“按任务选择一个服务器配置”，不是“按 item 选择不同服务器配置”。
