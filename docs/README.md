# 文档知识库总索引

> 文档定位：本文是 `docs/` 知识库总索引，只回答“事实去哪看”和“专题怎么下钻”，不承载项目协作规则正文。
>
> 事实依据：基于当前 `docs/` 目录结构与文档规范迁移计划整理。
>
> 权威入口：协作规则见仓库根目录 `PROJECT_SPEC.md`。
>
> 最后更新：2026-04-26

## 默认使用方式

1. 先读 `PROJECT_SPEC.md`，确认协作规则、验证边界和文档同步要求。
2. 再读本文，定位项目事实、方案、决策和历史材料。
3. 按任务进入对应目录的 `README.md` 和专题文档。

## 状态目录规则

新增计划、方案、审查等存在推进状态的文档时，在对应专题目录下使用状态子文件夹：

```text
未开始/
进行中/
已经完成/
```

文件名只表达主题与用途，不再使用 `未开始-`、`进行中-`、`已经完成-` 前缀。状态变化时移动文件，并同步更新本索引、目录 `README.md` 或相关专题索引。

## 历史材料使用方式

- `90-归档/` 中的文档用于追溯背景，不作为当前实现口径的唯一依据。
- 历史文档中的旧路径、旧命令或旧运行模式必须结合 `02-现状/`、`04-决策/` 与当前代码核实。
- 如果历史材料已经沉淀为当前口径，应优先回写当前有效文档，而不是继续在历史文档中追加流水账。
