# 文档目录说明

`docs/` 已按“总览 -> 现状 -> 方案 -> 决策 -> 审查与验证 -> 归档”重组，默认阅读顺序如下：

1. `01-总览/`：了解项目全局结构、逻辑文档索引和文本资源布局。
2. `02-现状/`：查看当前接口、项目结构和后端现状。
3. `03-方案/`：查看仍有参考价值的实施方案、专题计划和分阶段设计。
4. `04-决策/`：查看已经固化的边界说明、归属规则和判定模板。
5. `05-审查与验证/`：查看代码审查、回归检查和阶段性验证记录。
6. `90-归档/`：存放早期记录或不再作为主入口的历史文档。

## 目录结构

```text
docs/
  README.md
  01-总览/
  02-现状/
  03-方案/
  04-决策/
  05-审查与验证/
  90-归档/
  superpowers/
```

## 维护规则

- 新增“当前有效”的项目文档，优先放入 `01-05` 对应目录，不再直接散落在 `docs/` 顶层。
- 已完成、仅保留历史参考价值、或明显属于早期占位记录的文档，移动到 `90-归档/`。
- `docs/superpowers/` 保持原有 `plans / specs / checklists` 结构，视为独立子体系，不在本轮继续拆分。
- 若修改了实现基线、接口、结构、约束或验证方式，需要同步更新这里的导航和对应专题文档。
